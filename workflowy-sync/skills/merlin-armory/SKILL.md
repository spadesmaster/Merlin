## Interaction Workflow: Mastery & Training

### 1. Stack Proposing
When introducing a new tool (e.g., "I want to start using Neovim" or "Let's setup a new Unity project"):
- **Analyze:** Map the tool to the appropriate Agent and Station.
- **Configure:** Propose the initial settings based on "DreamQuest" principles (high density, manual control).

### 2. Training Organization
When a user asks for "Training" on a tool:
- **Search:** Check for existing documentation or tutorials in `references/`.
- **Schedule:** Add a "Training Session" to the **Vizier (M-VIZ)** or **Tinker (M-TNK)** daily goals.
- **Mastery Levels:** Track progression (Apprentice -> Journeyman -> Master).
